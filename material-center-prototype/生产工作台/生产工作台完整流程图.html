📋 数据字典

📦 任务数据 (Task)

  • id任务ID（唯一标识，自增）
  • name任务名称
  • creator创建人
  • createTime创建时间（yyyy-MM-dd HH:mm）
  • status状态：draft / processing / done
  • materials原料配置 {前贴:[], 中段:[], 尾帧:[]}
  • basicAttrs基础属性 {biz, channel, optimizer}
  • params生产参数 {resolution, framerate, bitrate, format}
  • namingParams命名规则 {fields:[]}
  • outputs成品列表 [{name, url, ...}]

🎬 原料数据 (Material)

  • id原料ID
  • name原料名称
  • type类型：前贴 / 中段 / 尾帧 / 中后段
  • biz业务类型
  • channel渠道
  • optimizer优化师
  • date上传日期
  • duration时长（秒）
  • size文件大小（MB）

⚙ 生产参数 (Production Params)

  • resolution分辨率：1080p / 720p / 4K
  • framerate帧率：25fps / 30fps / 60fps
  • bitrate码率：5 Mbps / 8 Mbps / 10 Mbps
  • format格式：MP4 / MOV / AVI

📝 命名规则参数 (Naming Params)

  • fields选中的命名字段数组
  • 原料名称使用的原料名称（可选）
  • 日期创建日期（可选）
  • 优化师优化师名称（可选）
  • 设计师设计师名称（可选）
  • 序号自动编号（可选）
  • 业务类型业务类型（可选）

状态说明

⏳ 草稿状态

  • 说明任务已创建但未开始生产
  • 允许操作编辑、删除、保存为草稿、开始生产
  • 校验规则不需要完整填写表单，名称为空自动填"未命名任务"

⚙ 生产中状态

  • 说明任务已提交生产，正在处理中
  • 允许操作监控生产进度、取消生产
  • 校验规则必须填写任务名称、至少选择一个原料
  • 成品数量根据原料组合自动计算（前贴×中段×尾帧）

✅ 已完成状态

  • 说明生产任务已完成，成品已生成
  • 允许操作导出成品、查看详情、删除
  • 导出格式支持批量导出为 ZIP

🔗 序列图（新建任务完整流程）

从左到右：用户 → 界面 → 逻辑层 → 数据存储

用户
点击"新建生产任务"
触发新建任务操作
界面
newTask()
清空表单，切换到配置视图（switchView('config')）
用户
输入任务名称
在任务名称输入框中输入内容（草稿模式可跳过）
用户
选择业务类型/渠道/优化师
填写基础属性，触发 updateBasicAttrs()
逻辑层
updateBasicAttrs()
更新 gBasicAttrs 全局变量
用户
点击"选择原料"
打开原料选择弹窗（openMaterialModal()）
逻辑层
openMaterialModal()
读取 gMaterials 原料库，渲染弹窗表格
用户
筛选并勾选原料
可按类型/业务/渠道/优化师/日期筛选，勾选所需原料
逻辑层
confirmSel()
保存已选原料到 gCurrentMaterials，关闭弹窗，显示已选标签
用户
配置生产参数
设置分辨率、帧率、码率、格式（可选，有默认值）
用户
配置命名规则
勾选命名字段（原料名称、日期、优化师、序号等）
用户
点击"保存为草稿"
不需要完整填写，直接保存当前配置
逻辑层
saveDraft()
任务名称空则填"未命名任务"，status=draft，写入 gTasks
界面
renderTaskList()
切换回列表视图，刷新任务列表，显示新草稿
用户
点击"开始生产"（草稿或新建页）
提交生产任务
逻辑层
saveTask(shouldStartProduction=true)
校验：任务名称非空、至少选一个原料；通过后 status=processing
逻辑层
生成成品（根据原料组合计算）
前贴×中段×尾帧 = 成品数量，按命名规则生成名称
逻辑层
更新任务 status=done
成品写入 gTasks[id].outputs，任务列表可查看/导出
