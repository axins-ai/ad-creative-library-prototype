状态说明

⏳ 草稿 draft

  • 说明任务已创建但未开始生产
  • 允许操作编辑、删除、保存为草稿、开始生产
  • 校验不校验表单完整性，名称空则自动填"未命名任务"

⚙ 生产中 processing

  • 说明任务已提交，正在处理
  • 校验必须填写任务名称 + 至少选1个原料
  • 成品数前贴 × 中段 × 尾帧 自动计算

✅ 已完成 done

  • 说明成品已生成
  • 允许操作导出成品、查看详情、删除
  • 导出支持批量导出 ZIP

🔗 序列图（新建任务完整流程）

用户 → 界面 → 逻辑层 → 数据存储 之间的交互时序

sequenceDiagram actor U as 用户 participant V as 界面 participant L as 逻辑层 participant D as 数据存储 U->>V: 点击「新建生产任务」 V->>L: newTask() L-->>V: 清空表单，切换配置视图 U->>V: 输入任务名称 U->>V: 选择基础属性（业务/渠道/优化师） V->>L: updateBasicAttrs() L->>D: 保存基础属性 U->>V: 点击「选择原料」 V->>L: openMaterialModal() L->>D: 读取原料库 D-->>L: 返回原料列表 L-->>V: 渲染原料弹窗 U->>V: 筛选并勾选原料 U->>V: 确认选择 V->>L: confirmSel() L-->>V: 关闭弹窗，显示已选标签 U->>V: 配置生产参数 U->>V: 配置命名规则 alt 保存为草稿 U->>V: 点击「保存为草稿」 V->>L: saveDraft() L->>D: 写入任务（status=draft） L-->>V: 切回列表视图 V-->>U: 显示新草稿任务 else 开始生产 U->>V: 点击「开始生产」 V->>L: saveTask(production=true) L->>L: 校验名称+原料 alt 校验不通过 L-->>V: 返回错误信息 V-->>U: 提示补全信息 else 校验通过 L->>D: 更新任务（status=processing） L->>L: 按原料组合生成成品 L->>D: 写入成品数据（status=done） L-->>V: 切回列表视图 V-->>U: 显示已完成任务 end end

📋 数据字典

📦 任务数据 Task

  • id任务ID（自增）
  • name任务名称
  • creator创建人
  • createTime创建时间
  • statusdraft / processing / done
  • materials{前贴:[], 中段:[], 尾帧:[]}
  • basicAttrs{biz, channel, optimizer}
  • params{resolution, framerate, bitrate, format}
  • namingParams{fields:[]}
  • outputs[{name, url, ...}]

🎬 原料数据 Material

  • id原料ID
  • name原料名称
  • type前贴/中段/尾帧/中后段
  • biz业务类型
  • channel渠道
  • optimizer优化师
  • date上传日期
  • duration时长（秒）
  • size文件大小（MB）

⚙ 生产参数

  • resolution1080p / 720p / 4K
  • framerate25 / 30 / 60 fps
  • bitrate5 / 8 / 10 Mbps
  • formatMP4 / MOV / AVI

📝 命名规则

  • fields选中的命名字段数组
  • 原料名称可选
  • 日期可选
  • 优化师可选
  • 设计师可选
  • 序号自动编号
  • 业务类型可选
